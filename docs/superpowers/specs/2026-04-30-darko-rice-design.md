# Darko Rice — Design Spec

**Date:** 2026-04-30
**Status:** Approved (pending user review of this document)
**Owner:** amir

## Overview

Replace the current monochrome ricing of the imperative-dots-based Hyprland setup with a full *Donnie Darko* — "cosmic dread" — visual theme covering every reachable surface (Hyprland chrome, popups, lockscreen, idle, GTK apps, Qt apps, file managers, terminals, SDDM, Plymouth, GRUB, wallpapers).

The theme is built as one of two coexisting variants (`mono` and `darko`) selectable at runtime via a `theme` command. The current monochrome rice becomes the `mono` variant; the new work is the `darko` variant. Live-reloadable surfaces switch instantly; reboot-required surfaces (SDDM, Plymouth, GRUB) are staged and take effect at next login or boot.

## Goals

- **Full Darko aesthetic** — palette plus literal motifs (Frank, "Wake Up", countdown, Sparkle-Motion-adjacent imagery) on visual surfaces.
- **Toggle-able** — `theme darko` and `theme mono` swap the live UI in <1s; staged surfaces follow on reboot.
- **No regressions** — current `mono` look is preserved 1:1 after the refactor.
- **Visual-only** — no audio assets, no easter eggs from other interests (Dota 2, Yu-Gi-Oh, The Office) — pure Darko.

## Non-goals

- Cursor recoloring (kept as Bibata-Modern-Classic, both themes).
- A wormhole-tunnel idle video loop (replaced by a 5s lavender dim → screen off).
- A Sparkle Motion login overlay (cut as fun-once novelty).
- Distributing the darko-pack as a public, portable plugin (private dotfiles only).
- Theming individual third-party app internals (Discord, Spotify, etc.) — they inherit GTK/Qt theming or stay default.

## Architecture

### Directory layout

```
~/.config/themes/
├── mono/
│   ├── matugen.toml           # palette seeds + extra hex constants
│   ├── manifest.json          # cursor=, icon=, gtk-name=, kvantum-name=
│   ├── wallpapers/            # rotation pool for this theme
│   ├── kitty.conf             # color block (matugen-rendered)
│   ├── kvantum.kvconfig       # Kvantum theme files
│   ├── gtk-colors.css         # gtk-4.0/colors.css (matugen-rendered)
│   ├── hyprlock.conf          # snippet
│   ├── hypridle.conf          # snippet
│   ├── mako.conf              # snippet
│   ├── sddm-theme/            # full SDDM theme dir
│   ├── plymouth-theme/        # full Plymouth theme dir
│   └── grub-theme/            # full GRUB theme dir
├── darko/
│   └── ...same structure...
└── active -> darko             # symlink — source of truth for current theme
```

The `active` symlink is the canonical answer to "what theme is on right now?" — every consumer (matugen build, scripts, status reporters) reads through it.

### `theme-switch` script

`~/.local/bin/theme-switch <mono|darko>` — also aliased as `theme`.

Behavior:

1. **Validate** target name; refuse if `~/.config/themes/<name>/` doesn't exist.
2. **Update** `~/.config/themes/active` symlink.
3. **Re-render** matugen using `active/matugen.toml` as the seed source. This regenerates all matugen-driven configs in their canonical paths.
4. **Live-reload** (no sudo, no reboot):
   - `hyprctl reload`
   - `pkill -SIGUSR2 quickshell` (or whatever the Quickshell reload signal is — to be confirmed during Phase 0)
   - `gsettings set org.gnome.desktop.interface gtk-theme adw-gtk3-dark`
   - `gsettings set org.gnome.desktop.interface icon-theme <from manifest>`
   - `gsettings set org.gnome.desktop.interface cursor-theme <from manifest>`
   - `kvantummanager --set <name from manifest>`
   - `makoctl reload`
   - kitty hot-reloads automatically via its config-watcher
   - kill+relaunch any running Dolphin process so it picks up the new Kvantum theme
5. **Stage** reboot-required surfaces (sudo prompt here, single time):
   - SDDM: `cp -r active/sddm-theme /usr/share/sddm/themes/active-darko-or-mono` and update `Current=` in SDDM config; effective on next login.
   - Plymouth: `cp -r active/plymouth-theme /usr/share/plymouth/themes/active`, `plymouth-set-default-theme active`, `mkinitcpio -P`. Skip the rebuild if `manifest.json` is unchanged from the last switch.
   - GRUB: `cp -r active/grub-theme /boot/grub/themes/active`, edit `/etc/default/grub` `GRUB_THEME=`, `grub-mkconfig -o /boot/grub/grub.cfg`.
6. **Report**:
   ```
   [live]    hyprland, quickshell, gtk, qt, kitty, mako, cursor, icons → darko
   [staged]  sddm (next login), plymouth + grub (next boot)
   ```
7. **No-arg invocation** (`theme`) prints the current `active` target.

### Failure modes

- Always `cp` the previous initrd to `/boot/initramfs-linux.img.bak` before `mkinitcpio -P` runs, so a broken Plymouth theme can't brick boot.
- Keep `/boot/grub/grub.cfg.bak` from last successful `grub-mkconfig`.
- Refuse to run if the target theme dir fails a quick syntax check (jq on `manifest.json`, file presence test on required artifacts).

## Lockscreen detail

Layout (rendered in hyprlock):

- **Background:** static gradient with two radial glows — `radial-gradient(ellipse 80% 50% at 50% 60%, rgba(122,74,138,0.25), transparent 60%)` plus `radial-gradient(circle at 20% 20%, rgba(245,160,64,0.08), transparent 40%)` over base `#0a0e1a`. No video; no GPU cost beyond hyprlock baseline.
- **Top corners:** small slate text — left `tangent universe // active` (uppercase, 11px monospace, letter-spacing 2px); right battery + wifi status.
- **Center:** large thin Helvetica clock (HH:MM, 96px, weight 200, letter-spacing 4px, lavender with subtle text-shadow). Above: "DARKO" wordmark, 14px serif, opacity 0.5, letter-spacing 8px. Below: weekday + date in serif, 13px, opacity 0.6.
- **Lower-center:** countdown widget — "END OF THE WORLD IN" (11px Courier, sodium amber, opacity 0.7) above a 32px Courier countdown `DD:HH:MM:SS` ticking down to `2026-05-28T06:42:12+02:00`. When it hits zero, reset to `now() + 28d 6h 42m 12s`.
- **Bottom-center:** "— wake up, donnie —" in 24px italic serif, opacity 0.85, with a thin-bordered password input below it (input field is transparent, bordered with `rgba(180,168,232,0.3)`, password chars rendered as `••••••••` in lavender Courier).

### Countdown implementation

hyprlock's native widget set may not handle `now() - target_date` math directly. Implementation plan:

1. **First-attempt:** declare a `text` widget with `cmd[update:1000]` that runs a tiny shell snippet computing the remaining time from `~/.config/themes/darko/end-date`.
2. **Fallback:** a long-running `darko-countdown.sh` daemon writes the formatted countdown to `/tmp/darko-countdown` once per second; hyprlock reads via `cat`.

Both are cheap; pick whichever is simpler given hyprlock's actual widget capabilities at implementation time.

## SDDM, GRUB, Plymouth detail

- **`darko-sddm`:** based on the existing `sddm-sugar-candy` style (a known-good Wayland-friendly QML SDDM theme). Backgrounded by `~/.config/themes/darko/wallpapers/sddm-bg.png`; login form pinned right-of-center, lavender `#b4a8e8` text on a `#1c2440` 0.6-alpha rectangle, sodium-amber `#f5a040` Login button.
- **`darko-grub`:** standard GRUB theme `theme.txt` format. Background: solid `#0a0e1a`. Title: top-left, sodium-amber serif, "DARKO". Menu: top-left below title, item text lavender, selected-item text sodium-amber. Bottom-right: 96×96 Frank silhouette PNG at 12% opacity. No icons.
- **`darko-boot` (Plymouth):** scripted Plymouth theme using the `script` plugin. ~90 lines of Plymouth-Lua. Single sequence: black screen → 15-frame fade-in of a 4px violet dot at (cx, cy) → 60-frame radial expansion to a 200px-radius soft disc with hard-edge fade-out → 30-frame fade to white → hand-off. Total runtime ~3.0s at 60fps.

## AI-gen image prompts

User runs these through their image generator and drops outputs into `~/Pictures/darko-source/`. We curate the keepers down to ~12 wallpapers + the SDDM bg + the Frank silhouette PNG.

### Wallpapers (16:9, 2560×1440 or higher)

1. **Suburban night street, sodium streetlamps, Virginia 1988.** Wide tree-lined road, no people, deep navy sky, amber lamp pools on wet asphalt. Cinematic film grain. Style: Richard Kelly cinematography, Steven Poster DP.
2. **Single sodium streetlamp on empty road**, low angle, lamp dominates frame, halo of warm orange in cold blue night.
3. **Tangent-universe portal effect** in a suburban living room — translucent, swirling, water-like vortex emerging from a chest. Violet-blue glow. No people.
4. **Cellar door at night**, slightly ajar, faint amber light from below, leaves on the ground. Eerie but quiet, not horror.
5. **Sky with a single jet engine** falling silently through high clouds, dawn light, no plane visible. Wide aspect, cinematic.
6. **Empty school auditorium**, dim, single spotlight on stage, dust motes, navy wash.
7. **Suburban house at 4am**, exterior, one upstairs window glowing amber, rest dark. Quiet menace.
8. **Black card with white serif text** "28:06:42:12" centered, large, no decoration. Pure typographic wallpaper.
9. **Abstract wormhole** — concentric violet rings receding into a black point, soft glow, no hard edges. Looks like space-time distortion.
10. **Wet asphalt close-up** with sodium streetlamp reflection, vertical bokeh, very moody.
11. **Lone hooded figure** at the end of a long suburban driveway, back to camera, sodium-lamp lit. Generic enough to not need a Frank costume.
12. **Bedroom interior at night**, single window with violet sky outside, no people, soft static light from an unseen TV.

### SDDM background

13. **Suburban street wide-shot at twilight**, full bleed, no central focal point (login form sits center-right of the rendered output, so leave that area visually quiet). Sodium-amber + cold-violet color grading.

### Frank silhouette (for GRUB)

14. **Solid black silhouette** of a tall thin humanoid figure with elongated rabbit ears, head and shoulders only, on a transparent background, 512×512 PNG. No facial detail, no texture — pure cutout. Used at 12% opacity in the GRUB corner.

### Lockscreen accent (optional)

15. **Soft tangent-universe haze** as a 2560×1440 PNG — abstract, all-violet-and-blue, no clear subject. Used as a subtle background under the lockscreen gradient if the static-gradient approach feels flat.

User generates a handful of variations per prompt, we pick the keepers together during Phase 4.

## Risks

| Risk | Likelihood | Mitigation |
|---|---|---|
| `mkinitcpio -P` fails, broken initrd → no boot | low | always `cp` previous initrd to `.bak` before rebuild; keep mono Plymouth staged as fallback |
| GRUB theme syntax typo → bootloader hangs | low–med | test in a chroot or VM first; keep `/boot/grub/grub.cfg.bak` |
| Matugen template breaks (seed value structure changed) | med | run `matugen --dry-run` after every template edit |
| AI-gen images come back unusable | med | iterate on prompts; fall back to user-provided source images for the failures |
| Hyprlock widget can't do `now() - end_date` math | med | fallback: `darko-countdown.sh` daemon writes to `/tmp/darko-countdown`, hyprlock `cat`s it |
| App stays in stale colors after switch (Dolphin specifically) | med | `theme` command kills + relaunches Dolphin if running |
| `papirus-folders --color violet` doesn't exist for our Papirus version | low | fall back to manually-overridden folder icons in `~/.icons/Papirus-Darko/` |
| Live-reload signals to Quickshell don't exist or differ from `SIGUSR2` | med | confirm during Phase 0; fall back to `pkill quickshell-bar && quickshell -p ... &` |
